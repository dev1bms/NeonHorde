## 4. Game Design Specification

### Core loop
Move with one thumb (floating virtual joystick — touch anywhere, drag). Weapons fire automatically. Kill glowing geometric enemies → they drop XP gems → level up → freeze: pick 1 of 3 upgrades → power compounds → beat PRIME to win. Death or victory → run summary → shards (meta currency) → permanent upgrades in the Upgrade Lab → next run starts stronger.

### Run structure & victory condition (exact)
Timer counts up. Elites at 2:30 / 5:00 / 7:30. **PRIME spawns at 9:00; regular spawns stop at 9:30. The run is WON at the moment PRIME dies — there is no survive-only victory, and the run does not end at 10:00 if PRIME lives. At 10:30 PRIME enrages: damage and speed +3% per second, compounding, so the fight cannot stall.** 10:00 is the design target for total run length, not a hard cutoff.

### Session shape & flows (exact)
- Typical death for new players: 2–4 min. Restart latency: < 1 s from death screen to new run.
- First-time UX: no tutorial screens; a 3-line hint overlay on first run only ("Drag to move — everything else is automatic").
- **Pause**: button top-right; tapping it, backgrounding, or any system/audio interruption freezes `World.tick` and shows an overlay: Resume / Settings toggles / Abandon Run (abandon banks shards earned so far). Level-up draft and pause are the only freeze states.
- **Death flow**: death slow-mo → run summary (time survived, kills, level, shards earned) → shards banked immediately and atomically → buttons: Retry (< 1 s), Upgrade Lab, Menu.
- **Shard formula** (initial, tunable in Balance.swift): elite/boss drops + `floor(secondsSurvived / 10)`.

### Enemies (all pure geometry, additive-blend neon glow)
| Name | Shape | Behavior | HP/Speed |
|---|---|---|---|
| Dart | small triangle | rushes straight at player | low / fast |
| Brick | square | slow relentless tank | high / slow |
| Splitter | pentagon | on death splits into 2 Darts | med / med |
| Weaver | diamond | sinusoidal approach (hard to lead) | med / fast |
| Spitter | hexagon | keeps distance, fires slow radial shots | med / slow |
| Elites | 2.5× scale, bright outline, at 2:30 / 5:00 / 7:30 | drop a chest = 1 guaranteed rare upgrade choice | very high |
| Boss: **PRIME** | huge rotating polygon | 3 phases: (1) charge dashes, (2) radial bullet rings + minion spawns, (3) enrage spin with beam sweeps | massive |

Spawn director: timeline-driven escalation (spawn rate, mix, HP multiplier all ramp). Spawns occur just off-screen, never within 150 pt of the player's heading.

### Weapons (8, each 5 levels; player holds max 4 weapons + 4 passives)
1. **Pulse Bolt** (starter) — auto-aims nearest enemy.
2. **Orbit Blades** — blades circle the player.
3. **Nova Burst** — periodic radial AoE ring.
4. **Rail Lance** — piercing beam through the densest cluster.
5. **Chain Arc** — lightning that jumps between enemies.
6. **Seeker Swarm** — homing micro-missiles.
7. **Mine Field** — drops proximity mines behind the player.
8. **Prism Beam** — rotating laser sweep (rare, high skill-feel).

Passives (each 5 levels): Damage %, Cooldown −%, Move Speed, Magnet Radius, Max HP, Armor (flat reduction), +Projectile Count, XP Gain %.

Level-up draft: 3 weighted random cards (owned items weighted up; never offer past max level; elite chests offer rare-tier picks with bigger jumps).

### Meta progression (Upgrade Lab, currency = Shards)
Permanent upgrades with escalating costs: Max HP, Base Damage, Move Speed, Magnet, Starting Level +1, Revive (max 1/run), Shard Gain. Cosmetic unlocks (shards): player core shapes (circle/triangle/star/hex) and trail colors.
**Overdrive** (unlocked after first victory): 5 tiers selected on the pre-run screen. Tier n: enemy HP ×(1+0.35n), enemy speed ×(1+0.06n), spawn rate ×(1+0.15n), shard gain ×(1+0.25n). Beating tier n unlocks n+1; stats screen tracks highest tier beaten. Constants live in Balance.swift and may be retuned.

### Balance targets (enforced by simulation tests)
- Fresh player (no meta, random-walk bot): dies between 1:30 and 4:00 (across 20 seeds).
- Bot with sensible auto-picked upgrades + kiting heuristic: reaches 6:00–9:00, usually dies to boss.
- Bot + 50% meta purchased: can beat PRIME.
- All tunables in one data file (`Balance.swift`) — tune there only. Assert on integer/ordinal outcomes (death tick, kill counts), never exact float equality.

### Juice checklist (all mandatory, Phase 7)
Screen shake (scaled by event), 40 ms hit-stop on big hits, white hit-flash on enemies, death slow-mo (0.3× for 0.5 s), pooled floating damage numbers (**pre-baked 0–9 digit glyph atlas composed from SKSpriteNodes — never SKLabelNode, which destroys batching**), player trail emitter, gem vacuum animation, level-up shockwave, boss intro banner + screen darkening, haptics (see Audio & Haptics).

### Art direction & rendering rules
Near-black background (#0A0A12) with subtle scrolling grid + parallax starfield; player cyan (#38F0FF), enemies magenta→orange ramp by threat (#FF3DAE → #FF7A1A), XP gems lime (#B6FF2E), UI white/cyan. Force dark appearance app-wide (`UIUserInterfaceStyle=Dark`) — this also makes the auto-generated launch screen black instead of white.
Sprite pipeline: draw each shape once at launch — **avoid `SKShapeNode.glowWidth` (inconsistent falloff); build glow from 2–3 stacked concentric shape layers with decreasing alpha, or bake a radial-gradient/blurred glow via CGContext** — capture with `SKView.texture(from:)` (main thread, on the live SKView, assert non-nil, prefer `texture(from:crop:)`), reuse in pooled `SKSpriteNode`s with `.add` blend mode. **Assign one zPosition band per entity type** (gems band, enemies band, projectiles band) so each type renders as one batch; verify with `showsDrawCount` in DEBUG. Keep glow sprite point-sizes modest (fill-rate on A11-class devices is the residual risk).

### Audio & Haptics (all generated by code — Phase 7)
- **SFX**: `tools/sfxgen` (a plain `swift` script) synthesizes sfxr-style sounds (laser, hit, explosion, pickup, level-up, UI tick, boss roar) by computing Float32 PCM samples directly and writing short `.caf` files via `AVAudioFile`. Playback: **one `AVAudioEngine` with a small pool of `AVAudioPlayerNode`s; all SFX pre-loaded into `AVAudioPCMBuffer`s at launch; `scheduleBuffer` per shot** (lowest latency, per-sound volume, one code path with music). Do not rely on `SKAction.playSoundFileNamed` (no volume control, memory quirks).
- **Music**: `tools/musicgen` renders a ~2:30 seamless dark-synthwave loop (minor-key bass + arpeggio + pad, ~110 BPM) plus a heavier boss layer — **encode to AAC `.m4a`** (a 2:30 stereo .caf is ~25 MB of PCM; unacceptable). Gapless looping: load into `AVAudioPCMBuffer`, schedule with `.loops` on an `AVAudioPlayerNode`. Crossfade layers by game intensity.
- **Machine-checkable audio gates** (the agent cannot listen): sfxgen/musicgen must assert — no clipping (|sample| < 1.0), loop-boundary sample continuity (first/last 64 samples cross-fade cleanly or match within epsilon), RMS within sane band, spectral energy present in expected bands. If generated music fails aesthetic judgment after 3 iterations, fall back to an intensity-driven ambient drone + rhythmic pulse — minimal beats bad.
- **Session**: `.ambient` category (never interrupts the user's own music); when `AVAudioSession.secondaryAudioShouldBeSilencedHint` (or `isOtherAudioPlaying`) is true, mute game music and keep SFX.
- **Haptics**: CoreHaptics — create the engine ONCE, guard with `CHHapticEngine.capabilitiesForHardware().supportsHaptics` (false on simulator → no-op), set `resetHandler`/`stoppedHandler` with lazy restart. Patterns: light tick = pickup, medium = hit taken, heavy = level-up/boss. If engine lifecycle proves fiddly after 3 attempts, fall back to `UIImpactFeedbackGenerator`/`UINotificationFeedbackGenerator`. Acceptance is via unit test (events map to correct pattern IDs) — physical feel is deferred to TestFlight and noted in RELEASE_RUNBOOK.md.
- Settings toggles (music/SFX/haptics) persisted in the save file.

Non-negotiable engineering rules:
- **Logic/render split**: `Core/` never imports SpriteKit → the whole game simulates headless at thousands of ticks/sec. Run the balance sweeps and soaks natively via `swift test` (macOS) — don't pay simulator boot cost per iteration.
- **Fixed timestep** (60 Hz accumulator) in `World.tick(input:)`; rendering interpolates. Determinism preconditions: seeded SplitMix64 only (never SystemRandomNumberGenerator), no wall-clock/Date in tick, never iterate Dictionary/Set where order affects outcomes (arrays + indices only).
- **Collision**: uniform-grid spatial hash (cell ≈ 64 pt), circle-vs-circle distance checks, preallocated flat buckets (no per-tick Dictionary/Set rebuilds). `SKPhysicsBody` is FORBIDDEN for gameplay entities (perf + non-determinism).
- **Pools everywhere**: fixed caps — enemies 600, projectiles 400, gems 300, damage numbers 60; node pools mirror them. Overflow policy: oldest gem merges, farthest enemy culled.
- **Swift language mode 5** (`SWIFT_VERSION: "5.0"` in project.yml — fully supported by Xcode 26's Swift 6.x toolchain; strict concurrency stays off).
- **Info.plist via XcodeGen's `info:` block** (NOT `GENERATE_INFOPLIST_FILE` — it can't express arbitrary keys). Properties: `UILaunchScreen: {}`, `UIUserInterfaceStyle: Dark`, `ITSAppUsesNonExemptEncryption: false`, `UISupportedInterfaceOrientations: [UIInterfaceOrientationPortrait]`, `UIStatusBarHidden: true`, `UIRequiresFullScreen: true`, `CFBundleDisplayName: Neon Horde`, **`NSPhotoLibraryAddUsageDescription`** ("Save your run's share card to Photos.") — without this, the share sheet's "Save Image" kills the app.
- **PrivacyInfo.xcprivacy** (required-reason APIs, avoids ITMS-91053 upload rejection): declare `NSPrivacyAccessedAPICategoryUserDefaults` reason `CA92.1` (if UserDefaults used anywhere) and `NSPrivacyAccessedAPICategoryFileTimestamp` reason `C617.1` (Application Support saves). Zero tracking, zero collected data types.
- Project generated by **XcodeGen** (`brew install xcodegen`; fallback: `git clone && swift build -c release`); never hand-edit the `.xcodeproj`. `TARGETED_DEVICE_FAMILY: "1"`, `MARKETING_VERSION: 1.0.0`, `CURRENT_PROJECT_VERSION: 1`.
